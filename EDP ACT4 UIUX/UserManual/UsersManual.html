User's Manual: Hospital Manangement System

1: Log In Page

The login process is a fundamental step that allows you to access the system and personalize your experience. Requires to input username and password.

If user don't have an account, You can click the add user accout to create yours.

2: Add User Account

In this section, The user is required to input his/her desired Username, Password and Confirm the password then click Add user button.

Click Cancel button if user changed his mind or to return to the log in page.

2: Dashboard

The dashboard page serves as a central hub within the system, providing users with an overview of important data and information.

Dashboard contains the following pages.
  • Add Table          - View Add table form and Back_Up action.
  • Check_Up         - View the Check_Up form.
  • Equipments       - View the Equipments form.
  • Patients             - View the Patients form.
  • Medicines          - View the Medineces form
  • Treatment Type - View the Treatment Type form.
  • Back to Log In   - Allows user to go back to Log in page upon clicking.
3: Add CSV File

In the Add CSV Form, the user can import a csv file and will be displayed in the datagrid of the form.

4: Save Back Up

Back Up button backs up the database and saves it in the chosen directory of the user.

5: Buttons

Utility Buttons, helps user to export,refresh and log out.

Print/Export button

Refresh button

Log Out button

  • The first button is print or export, this open the microsoft excel containing the template and table present in the form.
  • The second button is Refresh, it reloads the datagrid after performing and action such as adding records.
  • The third button is log out, After clicking the log out button the user will be returned to the log in page

Summary and Conclusion

Summary:

      The system is equipped with CRUD (Create, Read, Update, Delete) functionality and the ability to export data to Excel. Users are required to create an account and log in using their credentials. The system consists of various tables, including Add Table, Check Up, Equipments, Patients, Medicines, and Treatment Types. Users can perform actions such as adding, editing, deleting, and exporting data within these tables. The system is designed to be user-friendly and easy to navigate.

Conclusion:

      In conclusion, the user's manual provides a comprehensive guide to using the system effectively. By following the instructions outlined in the manual, users can take advantage of the system's CRUD capabilities and export data to Excel for further analysis. The login process ensures security and personalized access to the system. The different tables within the system allow users to manage and track various aspects such as check-ups, equipment, patients, medicines, and treatment types. Overall, the system aims to enhance productivity and streamline workflow organization in a user-friendly manner. With this user's manual, users will be equipped with the necessary knowledge to navigate and utilize the system efficiently.
